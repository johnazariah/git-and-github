# Git & GitHub for Research Collaboration

Welcome! This site hosts the materials for a **2-hour hands-on workshop** teaching STEM graduate students how to use Git and GitHub in their research.

---

## Before the Workshop

**[→ Complete the Pre-Workshop Setup Guide](pre-workshop/setup-guide.md)**

This takes about 15–20 minutes. Please do it before you arrive.

---

## Workshop Materials

### Exercises (follow along during the workshop)

1. **[Exercise 1: Create Your Research Repository](exercises/exercise-1-solo-repo.md)**
   Create a Git repo, make meaningful commits, and set up `.gitignore`.

2. **[Exercise 2: Research Team Collaboration](exercises/exercise-2-collaboration.md)**
   Pair up — branches, Pull Requests, code review, and merge conflicts.

3. **[Template Repo Files](https://github.com/johnazariah/git-and-github/tree/main/exercises/template-repo)**
   Starter files for the collaboration exercise.

### Handouts (take these home)

- **[Git Cheat Sheet](handouts/git-cheatsheet.md)** — Every command from today, plus research workflow patterns and notebook tips.
- **[GitHub Copilot One-Pager](handouts/copilot-one-pager.md)** — How to get free access and 5 prompts every researcher should know.

## Resources

### 📖 Essential Reading

| Resource | Why It's Useful |
|----------|----------------|
| [Pro Git Book](https://git-scm.com/book/en/v2) | The definitive Git reference. Free online. Chapters 1–3 cover everything from today. |
| [GitHub Skills](https://skills.github.com/) | Interactive courses that run in GitHub repos. Great for continued practice. |
| [Software Carpentry: Git](https://swcarpentry.github.io/git-novice/) | Git lesson designed specifically for researchers. |
| [The Turing Way](https://book.the-turing-way.org/reproducible-research/vcs) | Guide to reproducible research, including version control best practices. |
| [Happy Git for the useR](https://happygitwithr.com/) | Essential if you work in R. |

### 📓 Git + Jupyter Notebooks

Notebooks and Git have a complicated relationship. These tools help:

| Tool | What It Does |
|------|-------------|
| [nbstripout](https://github.com/kynan/nbstripout) | Auto-strips cell outputs before commit — keeps diffs clean |
| [Jupytext](https://jupytext.readthedocs.io/) | Syncs `.ipynb` with plain-text `.py` files for readable diffs |
| [nbdime](https://nbdime.readthedocs.io/) | Diff and merge tools built for notebooks |
| [ReviewNB](https://www.reviewnb.com/) | Visual notebook diffs in GitHub Pull Requests |

### 🎓 For Your Research

| Resource | What It Does |
|----------|-------------|
| [GitHub + Zenodo](https://docs.github.com/en/repositories/archiving-a-github-repository/referencing-and-citing-content) | Get a DOI for your repo — cite your code like a paper |
| [Choose a License](https://choosealicense.com/) | Simple guide to open-source licences for research code |
| [JOSS](https://joss.theoj.org/) | Publish your research software as a citable paper |
| [FAIR4RS](https://www.nature.com/articles/s41597-022-01710-x) | Why making code Findable, Accessible, Interoperable & Reusable matters |

### 🆘 When You're Stuck

| Resource | Best For |
|----------|----------|
| [Oh Shit, Git!?!](https://ohshitgit.com/) | Plain-English fixes for common Git mistakes |
| [Git Flight Rules](https://github.com/k88hudson/git-flight-rules) | Comprehensive FAQ — "I did X, how do I fix it?" |
| [Visual Git Reference](https://marklodato.github.io/visual-git-guide/index-en.html) | Diagrams showing what each command actually does |
| [GitHub Git Cheat Sheet (PDF)](https://education.github.com/git-cheat-sheet-education.pdf) | Print-friendly two-page command reference |

### 🔧 GitHub Features Worth Exploring

| Feature | How Researchers Use It |
|---------|----------------------|
| [GitHub Pages](https://pages.github.com/) | Publish project docs as a website (like this one!) |
| [GitHub Actions](https://docs.github.com/en/actions) | Automate tests, data validation, figure generation |
| [GitHub Organizations](https://docs.github.com/en/organizations) | Shared space for your lab group |
| [GitHub Discussions](https://docs.github.com/en/discussions) | Forum-style project Q&A |
| [GitHub Codespaces](https://github.com/features/codespaces) | Full dev environment in the browser — no local setup |
